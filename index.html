AndI Lua API Documentation

Welcome to the AndI Lua Scripting API. This document outlines all the built-in native functions available to your Lua scripts, allowing you to access the device's physical hardware, run local AI models, and manipulate the application's UI.

Tip: All functions execute synchronously in the Lua Isolate. However, heavy tasks (like llamaGenerate or openCamera) delegate their heavy lifting to the Main Dart Isolate to prevent your scripts from freezing.

🤖 Artificial Intelligence

llamaGenerate(prompt)

Prompts the locally loaded GGUF Llama model.

  • Parameters: prompt (String)
  • Returns: nil. The output is streamed asynchronously to the App Terminal.
log("Asking Llama...")
llamaGenerate("Write a haiku about a robot.")

geminiGenerate(prompt)

Prompts the Google Gemini API (requires network connection).

  • Parameters: prompt (String)
  • Returns: nil. The output is streamed asynchronously to the App Terminal.
geminiGenerate("What is the distance to the moon?")

📷 Camera & Computer Vision

openCamera(cameraIndex, [resolution])

Initializes a native camera stream and pipes it into the high-performance Double Buffer.

  • Parameters:
    • cameraIndex (Integer): The hardware camera index (usually 0 for rear, 1 for front).
    • resolution (String, Optional): "low", "medium", "high", "veryHigh", "ultraHigh", "max". Defaults to "medium".
  • Returns: nil
openCamera(0, "low") -- Fast 320p stream for CV processing

getCameraFrame()

Reads the latest frame from the native Double Buffer without blocking the UI.

  • Returns: A Table {width: Int, height: Int, format: Int, data: String} or nil if no frame is ready.
    Format: 0 for BGRA8888, 1 for YUV420.
while true do
  local frame = getCameraFrame()
  if frame ~= nil then
    log("Got frame: " .. frame.width .. "x" .. frame.height)
    -- Feed frame.data into an ONNX model!
  end
end

closeCamera()

Disposes the camera stream and frees resources.

closeCamera()

🧠 ONNX Machine Learning

onnxLoad(modelPath)

Loads an .onnx neural network file into the native C-Heap.

  • Parameters: modelPath (String) - Absolute file path.
onnxLoad("/path/to/model.onnx")

onnxRun(modelPath, inputJson)

Executes the ONNX model with the given input tensors.

  • Parameters:
    • modelPath (String) - Path of the previously loaded model.
    • inputJson (String) - JSON formatted tensor data.
  • Returns: nil. Output is sent to the App Terminal.
onnxRun("/path/to/model.onnx", '{"input_1": [[0.5, 0.2]]}')

onnxDispose(modelPath)

Safely unloads the ONNX model from the C-Heap to prevent memory leaks.

onnxDispose("/path/to/model.onnx")

📟 Hardware Sensors

Important: Hardware sensors drain battery. You must call initSensors() before reading data, and disposeSensors() when finished.

initSensors()

Wakes up the physical IMU chips and starts piping data into Native Memory.

initSensors()

getAccelerometer() / getGyroscope() / getMagnetometer()

Reads the latest XYZ vectors from the Native Shared Memory in O(1) time.

  • Returns: A Table { [1] = x, [2] = y, [3] = z }
initSensors()
while true do
  local acc = getAccelerometer()
  if acc[2] > 9.0 then
    setEmotion("happy") -- Phone is upright!
  end
end

disposeSensors()

Powers down the hardware streams.

disposeSensors()

🔌 Serial / USB Communication (AndI)

serialInit()

Initializes the USB-OTG connection to the connected microcontroller.

serialInit()

serialWrite(data)

Sends a string payload over the TX line.

  • Parameters: data (String)
serialWrite("M1: 255\n")

serialRead()

Synchronously drains the Native Lock-Free Ring Buffer for incoming RX bytes.

  • Returns: String containing the read bytes, or nil if the buffer is empty.
local data = serialRead()
if data ~= nil then
  log("Received from Arduino: " .. data)
end

📂 File System

readFile(path)

Reads a file from the device storage.

  • Parameters: path (String)
  • Returns: String contents, or nil if the file doesn't exist.
local text = readFile("/sdcard/config.txt")

listFiles(path)

Lists all files in a directory.

  • Parameters: path (String)
  • Returns: String of paths separated by newlines, or nil.

🎭 User Interface

setEmotion(emotion)

Instantly overrides the global RoboticEyeController emotion state. Built-in flood protection prevents UI crashing even if called in an infinite loop.

  • Parameters: emotion (String) - "normal", "angry", "happy".
setEmotion("angry")

showToast(msg)

Displays an Android-style Toast pop-up on the screen.

  • Parameters: msg (String)
showToast("Script Initialization Complete!")

log(msg)

Prints text directly to the AndI App's in-built script terminal.

  • Parameters: msg (String)
log("Variable value is: " .. x)
